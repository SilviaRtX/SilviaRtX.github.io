1. 样本概况

1.1 样本信息

样本名称 xiongmao.exe
文件大小 30001 bytes
修改时间 2018年7月14日, 8:40:21
MD5 512301C535C88255C9A252FDF70B7A03
SHA1 CA3A1070CFF311C0BA40AB60A8FE3266CFEFE870
CRC32 E334747C

1.2 测试环境及工具

测试环境：Windows7专业版32位

工具：

  • Hash-计算哈西
  • PCHunter32
  • 火绒剑
  • Procmon
  • PEiD
  • OD
  • x32dbg
  • IDA
  • 010Editor

1.3 样本行为

  • 创建spo0lsv.exe程序并位于C:\windows\system32\drivers目录
  • 将C:\Windows\system32\drivers\下程序spo0lsv.exe设为自启动
  • 关闭杀软服务与删除杀软启动项，关闭系统安全防护
  • 在各个磁盘根目录下复制自身并设置点击磁盘即自启动
  • 在大部分文件夹下创建文件Desktop_.ini，并写入感染日期
  • 将创建的文件设置为只读、系统、隐藏，并不可通过设置显示隐藏文件及文件夹显示出
  • 感染感染系统中exe、scr、pif、com、htm、html、asp、php、jsp、aspx等文件
  • 探测局域网内主机的139、445等网络文件共享端口，通过文件共享感染局域网。
  • 请求热门门户网站并发送数据，实现网络传播
  • 在命令行下解除各个磁盘和系统根目录下的文件共享。
  • 枚举进程、遍历窗口，关闭如任务管理器窗口等安全检测窗口。

2. 具体行为分析

2.1 主要行为分析

xiongmao.exe

1. 进程信息

此时病毒的行为可知：

  • 在C:\Windows\system32\drivers\下创建spo0lsv.exe程序并运行
  • 调用命令行使用net share命令删除各个磁盘及系统根目录共享

2. 文件信息

由图可知xiongmao.exe的主要行为是在C:\Windows\system32\drivers\下创建spo0lsv.exe

3. 分析小结

由上可知xiongmao.exe主要作用是在C:\Windows\system32\drivers\下释放程序spo0lsv.exe，即spo0lsv.exe才是病毒的主要文件。

spo0lsv.exe

1. 文件操作

此时病毒的行为可知：

  • 在大部分文件夹下创建文件Desktop_.ini，其内容为感染时间
  • 在各个磁盘根目录下创建文件autorun.inf和setup.exe,文件属性为系统、隐藏；其中autorun.inf的目的是当用户双击磁盘时自动运行setup.exe；setup.exe疑似病毒样本的拷贝。
  • 向可执行文件写入固定大小的30033字节，疑似病毒样本（30001）+ 32字节

2. 注册表操作

此时病毒的行为可知：

  • 病毒设置了自启动项，要启动的本体是drivers目录下的spoclsv.exe，每次开机就会自动运行这个病毒。
  • 对文件实现隐藏，设置该值后，即使我们在文件夹选项中选择显示所有文件和文件夹，也无法显示隐藏文件。综上可知对创建的文件Desktop_.ini、setup.exe、autorun.inf等隐藏了文件，即使设置显示隐藏文件及文件夹也无法显示出这些隐藏文件。
  • 网络设置，疑似有通过网络传染等行为。
  • 删除各大杀软的自启动和网络连接报错的自启动。

3. 网络操作

此时病毒的行为可知：

  • 探测局域网内主机的139、445等网络文件共享端口，疑似想要通过文件共享感染局域网。
  • 请求热门门户网站并发送数据，疑似测试网络环境或通过网页传播病毒。

4. 执行操作

此时病毒的行为可知：

  • 在命令行下解除各个磁盘和系统根目录下的文件共享。

5. 进程操作

此时病毒的行为可知：

  • 枚举进程，关闭窗口。在病毒运行中任务管理器窗口被关闭，为病毒隐藏自身的行为。

5. 分析小结

分析到这里，我们基本已经分析了熊猫烧香的病毒行为，但这些行为仍然无法彻底了解病毒的行为，还需要通过恶意代码分析来实现。目前病毒的行为有：

  • 在各个磁盘根目录下复制自身并设置点击磁盘即自启动
  • 在大部分文件夹下创建文件Desktop_.ini，并写入感染日期
  • 将创建的文件设置为只读、系统、隐藏，并不可通过设置显示隐藏文件及文件夹显示出
  • 感染可执行文件使其图标变为病毒图标，疑似在其中添加病毒程序和某个32字节大小的文件
  • 删除各大杀软的自启动以及网络连接报错自启动
  • 将C:\Windows\system32\drivers\下程序spo0lsv.exe设为自启动
  • 探测局域网内主机的139、445等网络文件共享端口，通过文件共享感染局域网。
  • 请求热门门户网站并发送数据，实现网络感染
  • 在命令行下解除各个磁盘和系统根目录下的文件共享。
  • 枚举进程、遍历窗口，关闭如任务管理器窗口。

2.2 恶意代码分析

通过前面的行为分析了解了病毒的主要行为，现在开始恶意代码分析，从反汇编代码中验证已知行为并判断有无其他行为。

这部分需要的工具有：

  • PEiD：查壳、程序版本信息
  • x32dbg：脱壳、修复导入表
  • IDA：静态分析
  • OD：动态验证分析

分析对象：xiongmao.exe的脱壳程序xiongmao_dump_SCY.exe

2.2.1 脱壳

由图可知本程序有壳，需要脱壳。发送到在x32dbg中脱壳后得到程序：xiongmao_dump_SCY.exe。

对脱壳后的程序用PEiD查看，可知其编程语言为：Borland Delphi 6.0 - 7.0

2.2.2 IDA静态分析和OD动态分析

1. 前期准备

载入Borland Delphi 6.0 - 7.0库文件

通过分析可知主程序中有四个关键函数:

  • sub_405250
  • sub_40819C
  • sub_40D18C
  • sub_40D088

2. 关键函数分析

1. sub_405250

进入OD动态验证分析

由此可知sub_405250是解码函数，用于验证程序是否为作者编写的病毒程序，解码字符串：xboy、whboy。

根据目前的信息在IDA中将此程序改名为DecodeString。

2. sub_40819C

1. 获取程序所在目录和程序名

2. 判断该文件夹下有无Desktop_.ini文件

3. 删除该文件夹下的Desktop_.ini

4. 读取病毒数据到内存

5. 判断当前程序是否为感染文件

通过前面010的截图可知010原本程序大小为7061336字节，被感染后病毒将该程序复制到末尾并添加被感染标识01。

由上可知当应用程序为感染文件时，病毒会还原原文件。通过验证发现只要运行了被感染程序，程序就会恢复正常。

6. 分析一个感染文件

目标文件：被感染的Hash.exe

由此可知被感染程序的行为有：

  • 释放文件Hash.exe.exe,查询当前文件下有无Hash.exe若有则替换该文件（删除、重命名）然后执行，并在系统驱动目录下创建病毒主体文件spo0lsv.exe并执行。

7. 函数功能总结

​ 分析到此基本可以确定本函数的功能如下:

  1. 判断当前病毒路径下是否有Desktop_.ini文件，若有则删除
  2. 读取病毒信息到内存
  3. 判断当前病毒类型（原始病毒程序?被感染的应用程序？病毒主体spo0lsv.exe？）
  4. 若为被感染的应用程序：判断当前有无病毒运行，若无则重新释放病毒主体并运行
  5. 若为原始病毒程序，则释放病毒主体spo0lsv.exe后结束
  6. 若为病毒主体则进入下一函数执行。

备注：每次病毒重新释放时都会删除一次系统驱动目录下的spo0lsv.exe以确保执行的是当前程序释放的病毒主体程序。

3. sub_40D18C

确认是驱动目录下的spo0lsv.exe后进入本函数。将样本改名为spo0lsv.exe后放到系统驱动目录下，再发送到OD中跟踪。

本函数中有三个函数：

  • sub_40A5B0
  • sub_40C374
  • sub_40BACC

**sub_40A5B0:**创建线程遍历目录创建Desktop_.ini

sub_40C374：设置一个定时器,在磁盘根目录下创建setup.exe和autorun.inf

sub_40BACC：创建10个线程，用于网络连接扩散病毒

4.sub_40D088

分析如下六个定时器函数：

  • sub_40CEE4
  • sub_40D040
  • sub_40D048
  • sub_407430
  • sub_40CC4C
  • sub_40C728

sub_40CEE4：修改注册表

sub_40D040：访问购物网站、下载恶意代码

回调函数：sub_40C9B0

sub_40D048：下载恶意代码并删除文件共享

sub_407430：删除杀软启动项及关闭安全相关服务

sub_40CC4C：访问解码后的url

sub_40C728：下载恶意代码

5. 感染函数

查看字符串得到一些常见的文件扩展名

经分析找到感染exe函数：sub_407F00

2.3 行为分析总结

熊猫烧香这一病毒具有典型的蠕虫病毒特征，通过多种途径传播感染。广域网方面通过访问热门网站疑似网页挂马传播，局域网上通过暴力枚举139、445等文件共享端口感染局域网。系统上通过创建隐藏文件、添加自启动项、感染文件等方式传播。同时为了自我保护，通过枚举进程、遍历窗口等方式关闭病毒查杀攻击、安全监测工具等服务并删除其注册表启动项。

3. 解决方案

3.1 提取病毒特征

​ 可将样本的MD5、SHA1、CRC32等提取为病毒特征，同时也可将程序中敏感字符串提取为病毒特征如xboy、WhBoy、*武汉*男生感染下载者**等。

3.2 查杀思路

  • 断开网络
  • 结束病毒进程
  • 删除C:\Windows\system32\drivers\下程序spo0lsv.exe
  • 删除磁盘下setup.exe与autorun.inf
  • 删除注册表启动项
  • 编写工具恢复被感染文件

4. 分析总结

对于不同的病毒应该注意不同的侧重点，如蠕虫病毒需要分析怎么传播，和主要行为；勒索病毒需要分析怎么加密勒索的和主要加密行为；
木马病毒，需要分析怎么藏匿的和主要行为以及通信协议。

本次分析中熟悉了IDA和OD的动静结合分析，同时深刻体会到Delphi程序寄存器传参的特点。Delphi在函数调用时参数的传递不完全用栈，主要用寄存器。而C++程序函数调用前会使用push语句将参数入栈，然后再进行call。Delphi一般将第一个参数放入eax寄存器，第二个参数放入edx，第三个参数放入ecx寄存器，其余参数按照与VC程序类似的方式压栈。同时在Delphi中字符串的首地址减去4后，所取出的4个字节的内容就是此字符串的长度。在分析中需要对一些语言特性方面的知识有所了解。
